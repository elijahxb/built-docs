Accessing document values in Painlessedit

The doc APIedit

The doc variable is available in search contexts that operate on individual documents. It exposes two methods from the Java Map interface, get and containsKey. In painless, Maps can use either the square brace or field syntax for working on fields, ie doc[myfield] or doc.myfield. If the field has dots in it’s name, the script must use square brace syntax.

doc has a key for each field in the mapping. The value is always a list, which may be empty if a document has no field data (doc values) for that field. The field data is the internal lucene representation, so, for text fields, it’s a sorted list of tokens. Scripts can use .value instead of [0] to access the first value of a field.

  • exposes doc values directly
  • .value shortcut
  • requires doc_values: true
  • doc values is always a List
  • key for every mapping

  • map-like Supported:

    • List get(Object key)
    • boolean containsKey(Object key) Unsupported:
    • int size()
    • boolean isEmpty()
    • boolean containsValue(Object value)
    • List put(String key, ScriptDocValues<?> value)
    • List remove(Object key)
    • putAll(Map)
    • void clear()
    • Set keySet()
    • Collection values()
    • Set<Map.Entry<String, ScriptDocValues<?>>> entrySet()
  • dot syntax directly works directly

Some documents may be missing a value for a field, so it is recommended

if (doc['myfield'].size == 0) {
  return;
}

If your script may run against multiple indices, some may not have mappings for the field, so it would need to check if the doc has the key as well.

if (!doc.containsKey('myfield') || doc['myfield'].size == 0) {
  return;
}
