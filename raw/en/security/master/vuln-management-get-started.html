Get started with CNVMedit

This page explains how to implement Cloud Native Vulnerability Management (CNVM) by configuring the Elastic Vulnerability Management for Cloud (EVMC) integration.

Requirements

  • CNVM is available to all Elastic Cloud users. On-premise deployments require an Enterprise subscription.
  • Elastic Stack version 8.8 or greater.

  • To view vulnerability scan findings, you need access to the following indices:

    • [TBD]

EVMC currently supports only AWS EC2 Linux workloads.

Set up EVMC for AWSedit

To set up the EVMC integration for AWS, install the integration on a new Elastic Agent policy, sign into the AWS account you want to scan, and run the CloudFormation template.

Add the EVMC integrationedit
  1. Go to the Elastic Security Get started page, and click Add security integrations.
  2. Search for Elastic Vulnerability Management for Cloud, then click on the result.
  3. Click on "Add Elastic Vulnerability Management for Cloud".
  4. Give your integration a name that matches its purpose or the region of the AWS account you want to scan for vulnerabilities, for example, uswest2-aws-account.
  5. Click Save integration. This integration will create a new elastic agent policy.
  6. Click Add Elastic Agent to your hosts.

Do not attempt to add the integration to an existing Elastic Agent policy. It should always be installed to a new policy, since it should not run on your VMs with existing workloads. For more information refer to Vulnerability Management Overview.

Sign in to the AWS management consoleedit
  1. Sign into your AWS management console in a new browser tab.
  2. Switch to the cloud region with the workloads which you want to scan for vulnerabilities.
Run the CloudFormation templateedit
  1. Switch back to the tab where you have Kibana open.
  2. Click Launch CloudFormation.
  3. Wait for the confirmation of Elastic Agent enrollment.
  4. Your data will start to appear on the Vulnerabilities tab of the Findings page.
