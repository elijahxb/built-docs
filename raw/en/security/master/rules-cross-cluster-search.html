Cross-cluster search and detection rulesedit

Cross-cluster search is an Elasticsearch feature that allows one cluster (the local cluster) to query data in a separate cluster (the remote cluster). Elastic Security’s detection rules can perform cross-cluster search to query security data in remote clusters, but it requires additional configuration and considerations.

  • Both the local and remote clusters must have the same role privileges to access the indices being queried on the remote cluster.
  • If there are any changes made to the role on the local cluster, you must update the API keys for all detection rules using that role for cross-cluster search. You can do this in Kibana → Stack Management → Rules on the local cluster. Refer to Update API keys for rules below for instructions.

Update API keys for rulesedit

If the role on the local cluster is changed, you’ll need to update the API keys for detection rules that use that role.

  1. On the local cluster, go to Kibana → Stack Management → Rules.
  2. Use the search box and drop-down filters to find the rules you want to update. For example, you can use the Type filter to find rules under the Security category.

  3. Select the checkbox next to each rule’s name, then Selected x rules → Update API keys.

    You can also update a single rule by selecting the rule’s actions menu (…​) → Update API key.
